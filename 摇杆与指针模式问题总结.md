# 摇杆与指针模式：已知问题总结

本文档基于当前仓库代码（用户已撤销此前实验性补丁后的状态）与掌机实机测试反馈整理，便于后续继续修复时对齐上下文。

## 环境与目标

- 平台：Linux ARM 掌机，Wayland，无物理鼠标。
- 目标行为：左摇杆驱动引擎内逻辑指针；GameController 下 **A** 模拟左键；摇杆出死区进入指针模式，一段时间无摇杆活动后回到传统模式；实体十字键（D-pad）按下时退出指针模式并隐藏光标。

## 用户侧已观察到的现象（按主题）

### 1. 摇杆跟手与“回放式”移动

- 光标延迟、不跟手。
- 连续推摇杆时间越长，松手后光标沿先前弯曲轨迹继续移动越久（像在处理积压的轴事件队列）。
- 曾怀疑：SDL 轴事件队列 + 每事件驱动位移；当前代码已部分采用 `drivePointerByCurrentStick()` 读实时轴，但 `updatePointerByStick` 仍会在有位移时 `return mouseMoveEvent(&mevent)`，与循环末尾再次 `drivePointerByCurrentStick()` 组合，行为需结合 `WAIT_*` 模式再验证。

### 2. 播放过场 / UI 滚动动画时的卡顿

- 在背景 UI 滚动等动画播放时，若摇杆持续移动（尤其推满），动画明显停顿；摇杆停住或松开后动画才继续。
- **与代码的对应关系（高概率）**：
  - `runEventLoop()` 中，当 `drivePointerByCurrentStick()` 返回 `true` 时会 **直接 `return` 退出整个事件循环**（例如超时分支约 `L1366`，循环末尾约 `L1853`）。
  - `waitEventSub()` 仅调用一次 `runEventLoop()`（`ONScripter_event.cpp` 约 `L310–L316`）。若在等待定时动画（`ONS_TIMER_EVENT` / `timerEvent`）期间，摇杆驱动路径频繁 `return`，会 **提前结束本次等待**，上层逻辑可能表现为动画节拍被打断或“停住直到下次进入等待”。
  - 指针每步移动会 `flush(refreshMode() | REFRESH_CURSOR_MODE, &screen_rect)`（约 `L1271`），整屏刷新与 `timerEvent` / `proceedAnimation` 同线程竞争，会加重掉帧。

### 3. 剧情对话面板上 load / skip 等按钮：A 与 B 行为异常

- 光标已移到 **load / skip** 等按钮上时，按 **A** 被处理为 **推进下一句**（类似空白左键 / LCLICK）；按 **B** 反而能点到按钮，但 **光标立即被隐藏**（用户期望不应隐藏）。

**与代码的对应关系：**

1. **GameController 键位语义（`transControllerButton`，约 `L160–L188`）**  
   注释写明为 Nintendo 式对调：**物理 A → `SDLK_ESCAPE`**，**物理 B → `SDLK_RETURN`**。  
   指针模式下仅 **`SDL_CONTROLLER_BUTTON_A`** 走 `pointerLeftClick` → 合成左键（约 `L1692–L1696`、`L1712–L1717`）；**B 不会走 `pointerLeftClick`**，会落入 `transControllerButton` → `SDL_KEYDOWN` / `keyPressEvent` 的 **键盘确认** 路径（与 `WAIT_BUTTON_MODE` 下 Return 选中当前悬停项的逻辑一致）。  
   这与用户描述的「B 能点中、A 像推进」在语义上 **一致**：A 实际是“左键点击路径”，B 实际是“键盘 Return 选中路径”。

2. **`mousePressEvent` 左键分支（约 `L439–L445`）**  
   使用 **`current_over_button`**，且 **`mousePressEvent` 内不调用 `mouseOverCheck`**。悬停依赖此前 **`mouseMoveEvent` → `mouseOverCheck`**（约 `L383–L384`）。  
   若 `current_over_button` 与当前 `pointer_cursor_*` 不同步（例如最后一帧无位移未触发 `mouseMoveEvent`、或菜单打开时 `refreshMouseOverButton()` 用系统鼠标坐标重置悬停，见 `ONScripter.cpp` 约 `L1239–L1252`），则 A 触发的合成点击会得到 **`current_over_button == -1`**，从而走 **`LCLICK`**（约 `L442–L444`），表现为 **推进对话而非点中按钮**。

3. **光标在按 B 后被隐藏**  
   - **`updatePointerIdleMode`（约 `L1279–L1284`）**：超过 `pointer_idle_timeout_ms` 未更新 `last_left_stick_active_ms` 会从指针模式切回传统并 `flush` 隐藏光标。若摇杆已回中一段时间，仅移动光标到按钮上而未产生有效位移刷新计时，可能在 **按 B 前后** 触发空闲退出。  
   - **`exitPointerModeForPhysicalDpad`（约 `L1286–L1290`）**：若设备或驱动在 **B 键** 上误报或与 **十字键** 事件交织，理论上会退出指针模式；需结合实机日志确认。  
   - 按 B 走键盘路径时 **`pointerLeftClick` 未刷新 `last_left_stick_active_ms`**，若同时摇杆在死区内，更容易满足空闲超时。

### 4. 五秒无摇杆隐藏光标

- 曾反馈逻辑不生效或边界异常；当前代码在 **有 `dx`/`dy` 位移** 时更新 `last_left_stick_active_ms`（约 `L1261–L1262`），进入指针时在 `entering_pointer` 分支刷新（约 `L1234–L1237`）。若“无位移但摇杆仍出死区”的帧不刷新计时，行为需与产品预期对齐说明。

### 5. D-pad 退出指针模式

- 当前已实现：`JOYHATMOTION` 与 `SDL_CONTROLLER_BUTTON_DPAD_*` 调用 `exitPointerModeForPhysicalDpad()`（约 `L1554–L1558`、`L1684–L1690`）。若实机仍异常，需区分走的是 Joystick 还是 GameController 路径。

## 此前实验性修改说明（用户已撤销）

曾尝试：强去队列化（合并轴事件、优先处理 timer）、指针 `flush` 节流、`pointerLeftClick` 前强制 `mouseOverCheck`、指针模式下拦截 B 的键盘路径、`drivePointerByCurrentStick` 不提前 `return`、`WAIT_BUTTON_MODE` 下跳过空闲退出等。**用户反馈问题仍未解决并已撤销**，当前以仓库现网代码为准；后续修复应基于本文件现象与上述代码对照重新设计、小步验证。

## 建议的后续修复方向（供实现时参考）

1. **动画与等待不被摇杆打断**  
   评估去掉或收窄 `drivePointerByCurrentStick()` 导致的 **`runEventLoop` 提前 `return`**，或改为仅更新状态、由统一出口结束等待；并评估指针移动的 **`flush` 频率** 与脏区范围。

2. **A 键稳定命中 UI 按钮**  
   在合成左键 **DOWN/UP** 前用 **`pointer_cursor_x/y` 强制 `mouseOverCheck`**，或让 `mousePressEvent` 在指针来源下重算悬停；核对 **`refreshMouseOverButton`** 与逻辑指针是否长期不一致。

3. **A/B 与产品语义对齐**  
   若产品要求“物理 A = 确认/左键”，需明确是否与 `transControllerButton` 的 Nintendo 对调策略冲突，并决定是否仅在 **`INPUT_MODE_POINTER`** 下覆盖语义，避免破坏传统键鼠场景。

4. **按钮界面下光标不应误隐藏**  
   在 `WAIT_BUTTON_MODE`（或等价 UI）下暂停空闲退出，或在任何指针交互（含按键）时刷新 `last_left_stick_active_ms`。
